## オペレータとモーションの組み合わせ

Vimの編集が効率的な理由の一つが、**オペレータ + モーション**という考え方や。

オペレータ（操作）にモーション（移動）を組み合わせることで、「どこまで何をするか」を表現する：

```
[オペレータ] [モーション]
    d            w      → 次の単語を削除（Delete Word）
    c            $      → 行末まで変更（Change to end of line）
    y            y      → 現在行をコピー（Yank Yank）
```

### 主要なオペレータ

| オペレータ | 動作 |
|---|---|
| `d` | 削除（Delete）＋ レジスタに保存 |
| `c` | 変更（Change）＝ 削除してインサートモードに入る |
| `y` | コピー（Yank） |
| `>` | インデントを増やす |
| `<` | インデントを減らす |
| `=` | 自動インデント |
| `gq` | テキストを整形 |

### よく使う組み合わせ

```vim
dw    " 次の単語を削除
d$    " 行末まで削除（D でも同じ）
dd    " 行全体を削除
d3j   " 現在行から3行下まで削除

cw    " 次の単語を変更（削除してインサートモード）
c$    " 行末まで変更（C でも同じ）
cc    " 行全体を変更（S でも同じ）

yy    " 行全体をコピー（Y でも同じ）
y3j   " 現在行から3行をコピー

p     " カーソルの後に貼り付け
P     " カーソルの前に貼り付け
```

## テキストオブジェクト

Vimの強力な機能の一つが**テキストオブジェクト**や。「単語の中」「括弧の中」「引用符の中」といった意味的な範囲を指定できる。

テキストオブジェクトは**`a`（around = 含む）**または**`i`（inner = 中だけ）**と組み合わせて使う：

```vim
" カーソルが hello の中にある場合
diw   " hello を削除（Delete Inner Word）
daw   " hello とその前後のスペースを削除（Delete Around Word）
ciw   " hello を変更（カーソルがどこにあっても単語全体が対象）
yiw   " hello をコピー
```

### 括弧・引用符に対するテキストオブジェクト

```vim
" function foo(arg1, arg2) { return bar; }
" カーソルが括弧内にある場合

di(   " 括弧内を削除 → function foo() { return bar; }
da(   " 括弧ごと削除 → function foo { return bar; }
ci(   " 括弧内を変更（括弧を残して中身を変更）
yi(   " 括弧内をコピー

" 同様に使えるもの
di"   " ダブルクォート内を削除
di'   " シングルクォート内を削除
di`   " バッククォート内を削除
di{   " 波括弧内を削除（di} でも同じ）
di[   " 角括弧内を削除
dit   " HTMLタグ内を削除（Delete Inner Tag）
```

**実践例：**

```javascript
// const message = "Hello, World!";
// カーソルが "Hello, World!" の上にある時

ci"  → "の中を変更。"" だけ残ってインサートモードに入る
// これで const message = "" になり、新しい文字列を入力できる
```

## mini.aiによる拡張テキストオブジェクト

MiniMaxには `mini.ai` が含まれとる。これは標準のテキストオブジェクトを大幅に拡張してくれるモジュールや。

### 関数の引数

```vim
" function foo(arg1, arg2, arg3)
" カーソルがarg2の上にある時

cia   " 引数を変更（Change Inner Argument）
daa   " 引数とカンマを削除（Delete Around Argument）
```

### Tree-sitterを使ったオブジェクト

```vim
af   " 関数全体を選択（Around Function）
if   " 関数の本体を選択（Inner Function）
ac   " クラス全体を選択
ic   " クラスの本体を選択
```

### 次のオブジェクトを指定

```vim
" カーソルが行頭にある時でも、次の括弧を対象にできる
cin(  " 次の括弧内を変更（Change Inner Next ）
din"  " 次のダブルクォート内を削除
```

## mini.surroundで囲み文字を操作する

コードを書いていると「この文字列をダブルクォートからシングルクォートに変えたい」「この変数を括弧で囲みたい」という場面が多い。`mini.surround`がそれを解決してくれる。

### デフォルトのキーマップ

| キー | 動作 |
|---|---|
| `sa{motion}{surrounding}` | 囲み文字を追加（Surround Add） |
| `sd{surrounding}` | 囲み文字を削除（Surround Delete） |
| `sr{old}{new}` | 囲み文字を置換（Surround Replace） |
| `sf{surrounding}` | 囲み文字を検索してカーソル移動 |

### 実用例

```vim
" hello という単語の上にカーソルがある時
saiw"   " → "hello"  （単語をダブルクォートで囲む）
saiw(   " → (hello)  （単語を括弧で囲む）

" "hello" の上にカーソルがある時
sd"     " → hello   （ダブルクォートを削除）
sr"'    " → 'hello' （ダブルクォートをシングルクォートに変更）

" <div>content</div> でタグの中にカーソルがある時
sdt     " タグを削除 → content
srt<span>  " タグを変更 → <span>content</span>
```

## mini.commentでコメント操作

コードのコメントアウトもよく使う操作や。`mini.comment`が提供するキーマップを使えば快適にできる：

```vim
gcc    " 現在行をコメントトグル
gc{motion}  " モーション範囲をコメントトグル
```

**例：**

```vim
" JavaScriptファイルで
gcc     " // コメント の追加・削除
gc3j    " 現在行から4行をコメントトグル

" ビジュアルモードで選択してから
gc      " 選択範囲をコメントトグル
```

## ドットリピート

Vimの隠れた強力機能が **`.`（ドット）**コマンドや。直前に行った変更操作を繰り返してくれる：

```vim
" 例: "foo"を'foo'に変換する作業が複数箇所ある場合

/foo     " まず一つ目を検索
sr"'     " ダブルクォートをシングルクォートに変更

n        " 次の "foo" に移動
.        " 同じ操作を繰り返す（sr"'が実行される）

n        " さらに次へ
.        " また繰り返す
```

`n`と`.`を交互に押すだけで、同じ変換を次々と適用できる。

## mini.moveで行を移動する

コードブロックを別の場所に移動したい時、`mini.move`が便利や：

```vim
" ノーマルモードで
Alt-j   " 現在行を1行下に移動
Alt-k   " 現在行を1行上に移動
Alt-h   " 現在行を左にインデント
Alt-l   " 現在行を右にインデント

" ビジュアルモードで選択してから
Alt-j   " 選択範囲を下に移動
Alt-k   " 選択範囲を上に移動
```

## mini.splitjoinで引数を整形する

関数の引数を一行から複数行に展開したり、逆にまとめたりできる：

```vim
" gS でトグル
function foo(arg1, arg2, arg3) {}
" ↓ gSを押すと
function foo(
  arg1,
  arg2,
  arg3,
) {}
" ↓ もう一度gSを押すと元に戻る
```

## レジスタの基本

`d`や`y`でコピーしたテキストは「レジスタ」に保存される。デフォルトは無名レジスタ（`"`）に入る。

```vim
" 特定のレジスタに保存
"ayy    " aレジスタに現在行をコピー
"adw    " aレジスタに次の単語を切り取り

" 特定のレジスタから貼り付け
"ap     " aレジスタの内容を貼り付け

" システムクリップボード（+レジスタ）
"+yy    " 現在行をシステムクリップボードにコピー
"+p     " システムクリップボードから貼り付け
```

:::message
MiniMaxのデフォルト設定では、`mini.basics`によってシステムクリップボードとの連携が有効になっとるので、通常の`y`と`p`でもクリップボードとやり取りできる環境が多い。
:::
