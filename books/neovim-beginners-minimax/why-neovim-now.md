## AIコーディング時代のエディタ選び

2024〜2025年にかけて、AIコーディングツールが爆発的に普及した。GitHub Copilot、Cursor、Claude Code、aider、Gemini CLI…エンジニアの開発スタイルが大きく変わりつつある。

特に注目したいのは、**ターミナルで動くAIツール**の台頭や。

```bash
# ターミナルで完結するAIコーディングの例
claude "このファイルのバグを修正して" --file src/app.ts

# aiderでインタラクティブにコード編集
aider src/app.ts

# AIが提案したコードをNeovimで確認・編集
nvim src/app.ts
```

AIがコードを生成・修正してくれる。でも最終的にコードを確認して、調整して、コミットするのは人間や。このワークフローにおいて、ターミナルで高速に動作するエディタがあると、ストレスなく作業できる。

さらに、SSHでリモートサーバーに接続しながら開発する機会も増えとる。そんな環境では、GUIのIDEは使えない。ターミナルエディタが唯一の選択肢になることもある。

## Neovimの何がすごいのか

### 圧倒的な起動速度と軽量さ

VSCodeを起動してプロジェクトを開くまでに数秒かかることがある。Neovimは0.1秒以下で起動する。

```bash
# 起動時間の比較（目安）
# VSCode: 2〜5秒
# Neovim (MiniMax): 0.05〜0.2秒
time nvim --headless +qa
```

メモリ消費も桁違いに少ない。VSCodeが300〜500MB使う場面でも、Neovimは30〜50MB程度で動く。

### リモート環境でもフル機能

SSH接続先でもローカルと同じ操作感で使える。これはターミナルエディタにしかできないことや。

```bash
# リモートサーバーでNeovimを使う
ssh user@remote-server
nvim src/main.go
# ローカルと全く同じキーマップ、LSP、補完が動く
```

### Vim/Neovimで学んだスキルは一生もの

Vimのキーバインドは他のエディタでも使える。

- **VSCode**: Vim拡張（[vscodevim](https://marketplace.visualstudio.com/items?itemName=vscodevim.vim)）
- **JetBrains系**: IdeaVim
- **Emacs**: Evil mode
- **ブラウザ**: Vimium
- **シェル**: `set -o vi`

一度身につければ、どのツールでも活かせる「筋肉記憶」になる。

### Neovim固有の強力な機能

Vimから派生したNeovimには、モダンな開発に欠かせない機能が揃っとる。

| 機能 | 説明 |
|---|---|
| **ネイティブLSP** | コード補完・定義ジャンプ・エラー表示をネイティブサポート |
| **Tree-sitter** | 高精度な構文解析によるシンタックスハイライト |
| **Lua設定** | 高速なLuaでカスタマイズ可能 |
| **非同期処理** | UIをブロックせずにバックグラウンド処理 |
| **組み込みターミナル** | エディタ内でAIツールやシェルを実行 |

## Vimの学習コストは本当に高いのか

「Vimは学習コストが高い」という評判があるけど、実際のところどうなんやろ。

最初の壁は確かにある。特に「モード」という概念——テキストを入力するモードと、コマンドを実行するモードが分かれているという考え方に慣れるまでは、戸惑うかもしれない。

でも本当に必要なキーバインドは思ったより少ない。この本を読み終えた後に日常開発で使うキーの9割は：

```
h j k l  → 移動
i a o    → 入力開始
Esc      → ノーマルモードに戻る
:w :q    → 保存・終了
/        → 検索
d c y p  → 削除・変更・コピー・貼り付け
```

この20個くらいを覚えれば、もう普通に使い始められる。

## なぜMiniMaxを選ぶのか

Neovimをゼロから設定するのは確かに大変や。そこで「スタータープロジェクト」を使うのが一般的なアプローチになっとる。

主な選択肢を比較してみよう。

| プロジェクト | 種類 | 特徴 |
|---|---|---|
| [LazyVim](https://www.lazyvim.org/) | ディストリビューション | 機能豊富、アップデートで設定も更新される |
| [NvChad](https://nvchad.com/) | ディストリビューション | 美しいUI、テーマが豊富 |
| [AstroNvim](https://astronvim.com/) | ディストリビューション | 高いカスタマイズ性 |
| [kickstart.nvim](https://github.com/nvim-lua/kickstart.nvim) | スタータースクリプト | シンプル、教育目的向け |
| **MiniMax** | **設定ジェネレータ** | **生成後は自分の設定に、mini.nvim統一** |

### MiniMaxの特徴

**1. 設定ファイルは「自分のもの」になる**

MiniMaxはセットアップ時に設定ファイルを生成する。生成後のファイルはあなたのリポジトリで管理できる。ディストリビューションのように上流の変更に振り回されない。

**2. mini.nvimで統一された体験**

MiniMaxはほぼ全機能をechasnovskiが単独で開発・メンテナンスしとる[mini.nvim](https://github.com/echasnovski/mini.nvim)のモジュール群で構成されとる。作者が一人なので設計思想が統一されとり、各プラグイン間の相性問題が起きにくい。

**3. 豊富なコメントが学習教材になる**

生成された設定ファイルには、各設定の意味を説明するコメントが丁寧に書いてある。設定を変えながらNeovimを学んでいける。

**4. Neovimバージョン対応**

Neovim 0.10、0.11、0.12それぞれに最適化された設定が用意されとる。

:::message
「とりあえず動くNeovim環境を手に入れて、あとは自分でいじりたい」——そういう人にMiniMaxはぴったりや。
:::
