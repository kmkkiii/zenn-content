## ファイルとバッファの概念

Neovimでは「ファイル」と「バッファ」を区別することが大事や。

- **ファイル**: ディスクに保存されているもの
- **バッファ**: Neovimがメモリに読み込んでいるもの

ファイルを開くとバッファが作られる。複数のファイルを開くと複数のバッファが存在する。バッファは閉じるまでメモリに残るので、一度開いたファイルは素早くアクセスできる。

さらに「ウィンドウ」と「タブ」もある：

- **ウィンドウ**: バッファを表示する「画面の枠」。同じバッファを複数のウィンドウに表示することもできる
- **タブ**: ウィンドウのレイアウトをまとめたもの

この概念を理解しておくと、ナビゲーション操作がスムーズになる。

## mini.filesでファイルを管理する

`mini.files`はMiniMaxに含まれるファイルエクスプローラや。ファイルツリーを表示して、ファイルの作成・削除・リネーム・移動ができる。

### mini.filesを開く

MiniMaxのデフォルトキーマップ：

```vim
<Space>e   " ファイルエクスプローラを開く（現在のファイルのディレクトリ）
<Space>E   " プロジェクトルートでファイルエクスプローラを開く
```

### ナビゲーション操作

```vim
" mini.files内での操作
h          " 親ディレクトリに移動
l          " ディレクトリを開く / ファイルを開く
j k        " 上下に移動
Enter      " ファイルを開く
q          " mini.filesを閉じる
<Esc>      " 閉じる
```

### ファイル操作（ファイルシステム操作）

`mini.files`の便利な点は、**ファイルを編集するようにファイルシステムを操作できる**ことや。

```vim
" mini.files内で
a          " 新しいファイル/ディレクトリを作成
r          " リネーム
d          " 削除
```

:::message
`mini.files`では、ファイル名を直接テキストとして編集できる。`ciw`でファイル名を変更してEnterを押すとリネームされる。直感的で Vim らしいアプローチや。

実際の変更は`=`（同期）を実行するまでディスクに反映されない。間違えても`q`で閉じれば変更を取り消せる。
:::

### プレビュー表示

```vim
<Tab>      " プレビューを表示/非表示
```

ファイルにカーソルを置いて`<Tab>`を押すと、右側にプレビューが表示される。

### 隠しファイルの表示

```vim
g.         " 隠しファイル（ドットファイル）の表示トグル
```

## mini.pickでファジー検索する

`mini.pick`はMiniMaxのファジーファインダーや。あいまいな文字列でファイルや内容を素早く検索できる。

### 主要なピッカー

MiniMaxでは`<Space>f`プレフィックスでpick系コマンドが集まっとる：

```vim
<Space>ff  " ファイルを検索（Find Files）
<Space>fb  " バッファを検索（Find Buffers）
<Space>fg  " ファイル内テキストを検索（Find Grep）
<Space>fh  " ヘルプタグを検索（Find Help）
<Space>fr  " 最近使ったファイルを検索（Find Recent）
```

### ファジー検索の使い方

`<Space>ff`でファイル検索を開くと：

```
> src/
  src/components/Header.tsx
  src/components/Footer.tsx
  src/pages/index.tsx
  src/utils/api.ts
```

検索文字列を入力すると、ファジーマッチングでフィルタリングされる：

```
> hdr
  src/components/Header.tsx  ← "hdr"でマッチ
```

完全一致でなくても文字が含まれていればマッチする。

### mini.pick内の操作

```vim
Ctrl-n / Ctrl-j   " 次の候補へ
Ctrl-p / Ctrl-k   " 前の候補へ
Enter             " 選択してファイルを開く
Ctrl-s            " 水平分割で開く
Ctrl-v            " 垂直分割で開く
Ctrl-t            " 新しいタブで開く
<Esc>             " キャンセル
```

### grep検索（ripgrep連携）

`<Space>fg`でプロジェクト全体のテキスト検索ができる。ripgrepがインストールされていれば高速に動作する：

```vim
<Space>fg
" 検索ワードを入力: function getUserById
" マッチした行が一覧表示される
```

## バッファの操作

### バッファ間の移動

```vim
[b    " 前のバッファへ（mini.bracketed）
]b    " 次のバッファへ（mini.bracketed）
<Space>fb  " バッファ一覧からファジー選択
```

画面上部の `mini.tabline` にバッファの一覧が表示されとる。番号も表示されるので、その番号でも移動できる：

```vim
:b3   " 3番のバッファへ移動
```

### バッファを閉じる

```vim
<Space>bd  " 現在のバッファを閉じる（mini.bufremove）
:bd        " バッファを削除（ウィンドウレイアウトが変わることがある）
```

:::message
`:bd`を使うと開いているウィンドウレイアウトが変わってしまうことがある。`mini.bufremove`の`<Space>bd`を使うと、ウィンドウレイアウトを保ったままバッファを閉じられて便利や。
:::

## ウィンドウの分割

一つの画面に複数のファイルを同時に表示したい場面は多い。Neovimはウィンドウ分割が得意や。

### 分割コマンド

```vim
Ctrl-w s   " 水平分割（Split）
Ctrl-w v   " 垂直分割（Vertical split）
:sp ファイル名  " 水平分割で別ファイルを開く
:vsp ファイル名 " 垂直分割で別ファイルを開く
```

### ウィンドウ間の移動

```vim
Ctrl-w h   " 左のウィンドウへ
Ctrl-w j   " 下のウィンドウへ
Ctrl-w k   " 上のウィンドウへ
Ctrl-w l   " 右のウィンドウへ
```

### ウィンドウのサイズ変更

```vim
Ctrl-w =   " 全ウィンドウを等分
Ctrl-w _   " 縦方向に最大化
Ctrl-w |   " 横方向に最大化
Ctrl-w +   " 高さを増やす
Ctrl-w -   " 高さを減らす
Ctrl-w >   " 幅を増やす
Ctrl-w <   " 幅を減らす
```

### ウィンドウを閉じる

```vim
Ctrl-w q   " 現在のウィンドウを閉じる
Ctrl-w o   " 他の全ウィンドウを閉じる
```

## セッション管理（mini.sessions）

作業状態（開いているファイル、ウィンドウレイアウト等）をセッションとして保存して、後で復元できる：

```vim
:lua MiniSessions.write('my-project')   " セッションを保存
:lua MiniSessions.read('my-project')    " セッションを読み込み
:lua MiniSessions.delete('my-project')  " セッションを削除
```

:::message
mini.sessionsはプロジェクトごとにセッションを管理するのに便利や。朝から作業を始める時に`:lua MiniSessions.read('project-name')`を実行すれば、昨日の状態からすぐに再開できる。
:::

## ファイル訪問履歴（mini.visits）

`mini.visits`は最近開いたファイルの履歴を管理してくれる。`<Space>fr`で最近のファイルにアクセスできる他、よく使うファイルをラベル付けすることもできる：

```vim
<Space>fr  " 最近のファイル一覧（mini.pick経由）
```

## まとめ：効率的なナビゲーションの流れ

典型的な作業フロー：

```
1. プロジェクトを開く
   nvm .              " カレントディレクトリでNeovimを起動

2. ファイルを探す
   <Space>ff          " ファジー検索でファイルを開く

3. コードを探す
   <Space>fg          " grep検索でコードの場所を特定

4. ファイルツリーを操作
   <Space>e           " mini.filesでファイルを整理

5. 複数ファイルを参照
   Ctrl-w v           " 垂直分割で隣に別ファイルを開く

6. バッファを切り替え
   [b ]b              " 開いているファイル間を移動
```
